# 权限与审批矩阵

本文说明各角色可执行的操作，以及哪些变更需要审批。

## 角色

| 角色 | 说明 |
|------|------|
| **SUPER_ADMIN** | 全站配置、用户/主题/策略；变更通常**直接生效** |
| **API_EDITOR** | 加入主题后可编辑该主题下 API、连接串；多数写操作需**审批** |
| **API_VIEWER** | 只读查看有权限主题下的资源 |
| **THEME_ADMIN** | 主题成员角色，可审批本主题待办、管理成员与 Key |
| **THEME_MEMBER** | 可提交 API/连接串变更，**不能**删除 API/连接串 |

## 操作矩阵

| 操作 | 超管 | 主题管理员 | 主题成员 | 审批 |
|------|------|------------|----------|------|
| 创建/编辑主题、成员 | ✅ 直接 | ❌ | ❌ | 否 |
| 创建/轮换主题 API Key | ❌ 仅查看 | 提交后审批 | ❌ | 是 |
| 新建/更新连接串 | ✅ 直接 | 提交后审批 | 提交后审批 | 是（非超管） |
| **删除连接串** | ✅ 直接 | 提交后审批 | ❌ | 是（非超管） |
| 新建/更新 API、版本 | ✅ 直接 | 提交后审批 | 提交后审批 | 是（非超管） |
| 发布/暂停/恢复版本 | ✅ 直接 | 提交后审批 | 提交后审批 | 是（非超管） |
| **删除 API** | ✅ 直接 | 提交后审批 | ❌ | 是（非超管） |
| 网关限流/熔断策略 | ✅ 直接 | 只读 | 只读 | 否 |
| 监控大盘 / 访问日志 | 全站 | **仅本主题** | **仅本主题** | — |
| Legacy Consumer Key | 只读总览 | 不可见 | 不可见 | — |

## 审批规则

- **任一**其他主题管理员或超级管理员审批通过即生效（无多级链）。
- 同一资源同时只能有 **1 条**待审批单（`PENDING`）。
- 审批单中的连接串密码**不以明文展示**；通过后由系统从加密侧栏应用。
- 数据源/API 删除与创建/更新一样走审批（非超管）。

## Legacy Consumer 下线

- 目标模型：**每主题一把 API Key**，自动授权该主题已发布 API。
- 生产环境：`gateway.consumer.legacy-enabled=false`，且 `bootstrap-key` 为空。
- 计划在 `gateway.consumer.legacy-sunset-date`（默认 2026-12-31）前完成迁移。
- 「调用方 / Key」页展示 Legacy 占比与迁移指引。
